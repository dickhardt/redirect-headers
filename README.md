# HTTP Redirect Headers

This repository contains the IETF Internet-Draft for HTTP Redirect Headers.

## Draft Document

- **Latest draft**: [draft-hardt-httpbis-redirect-headers-00](draft-hardt-httpbis-redirect-headers-00.md)
- **Explainer**: [explainer.md](explainer.md) - Detailed explanation and use cases

## Abstract

This document defines HTTP headers that enable browsers to pass redirect parameters securely during HTTP redirects without exposing them in URLs. The Redirect-Query header carries parameters traditionally sent via URL query strings, the Redirect-Origin header provides browser-verified origin authentication, and the Redirect-Path header enables path-based redirect validation.

## Building the Draft

### Prerequisites

Install mmark via homebrew:
```bash
brew install mmark
```

### Local Build

Build XML and HTML for local preview:
```bash
make
```

View the HTML:
```bash
open draft-hardt-httpbis-redirect-headers-00.html
```

**Note**: Local HTML generated by mmark is for content preview only. Author information and full IETF formatting will appear in the official outputs generated by GitHub Actions.

### Available Make Targets

- `make` or `make all` - Build XML and HTML
- `make xml` - Build only XML
- `make html` - Build only HTML
- `make clean` - Remove generated files
- `make help` - Show all targets

## Automated Builds

GitHub Actions automatically builds official IETF-compliant XML, HTML, and TXT formats on every push to main. Artifacts are available in the Actions tab.

## Submitting to IETF

1. Download the XML or TXT file from GitHub Actions artifacts
2. Go to https://datatracker.ietf.org/submit/
3. Upload the file and follow the submission process
4. Authors will receive email confirmation requests
